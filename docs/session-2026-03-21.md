## Architecture decisions made

- **Two set types:** `main` (rarity rules) and `special` (manual override sheet required)
- **`.5` in set ID = special set** — rule is enforced in `create-input-workbook.py`
- **Bulbapedia is prep-only** — used by `prep-special-set.py` to draft overrides, never at pipeline runtime
- **One image per card**, reused across all variants (no separate image per variant)
- **Unknown rarity defaults to no reverse holo** — safe fallback
- **Special sets block the pipeline** if override entries are missing or unreviewed
- **Master catalogue is pure reference data** — inventory, listings, pricing live in separate workbooks (future)
- **Pricing:** TCGPlayer USD from TCGdex API, converted to AUD via Frankfurter. `price_aud_ebay` column reserved for Phase 2 (eBay AU API — developer account exists)

## Image research finding

TCGPlayer CDN hosts card images at predictable URLs using product IDs:
```
https://tcgplayer-cdn.tcgplayer.com/product/{PRODUCT_ID}_in_1000x1000.jpg
```
The TCGdex API returns `pricing.tcgplayer.normal.productId` per card — so we could construct TCGPlayer image URLs without scraping. Worth exploring as a higher-quality image source than TCGdex.

No free source exists for reverse holo variant images. Physical photography is the only accurate option.
