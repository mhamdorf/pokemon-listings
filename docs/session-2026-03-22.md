## What was shipped

### `scripts/build-stock-sheet.py`

Seeds/updates the Stock sheet in `data/input/ebay.xlsx` from the master catalogue. One row per card variant with Qty + Price columns (both user-editable, yellow). Preserves existing values on re-run — only adds new rows.

Run once per set after it's in the catalogue:

```bash
uv run python scripts/build-stock-sheet.py --set me02.5 --max-card 217
```

### `scripts/export-ebay-csv.py`

Generates a Seller Hub multi-variation listing CSV. One parent row (listing metadata) + one child row per in-stock variant. Pricing falls back to `--price-normal/holo/rh` defaults unless a per-row `Price` is set in `ebay.xlsx`.

```bash
uv run python scripts/export-ebay-csv.py \
    --set me02.5 \
    --title "✅ PICK YOUR CARD ✅ ASCENDED HEROES — Pokémon TCG Singles NM FREE POST" \
    --location "Adelaide, SA" \
    --price-normal 1.00 \
    --price-holo 1.25 \
    --price-rh 1.50 \
    --shipping-profile "Australia Post Domestic Regular Letter Untracked - TCG" \
    --return-profile "No Returns - TCG"
```

Upload via: **Seller Hub → Reports → Upload → Create listings**

### `docs/how-to/create-listing-view.md`

Power Query M query for the Listing view in `ebay.xlsx`. Joins Stock sheet to catalogue, expands variants, filters to Qty ≥ 1.

## Known limitations / decisions made

### Per-card image swapping doesn't work via CSV upload

eBay's "Variation Specific Picture Set" (where the gallery image swaps when a buyer selects a card from the dropdown) can only be set through the **eBay API** or manually in the listing editor — not via File Exchange CSV upload. The `PicURL` on child rows in a CSV is treated as additional listing-level images, not variation-mapped images.

**Current state:** Single cover photo (Ascended Heroes artwork) shows for all cards. This is normal for pick-your-card listings.

**To fix properly:** Use the eBay Inventory API to create/revise listings programmatically, which supports `VariationSpecificPictureSet`. Worth tackling when scaling to multiple sets.

### Stock management workflow

1. Run `build-stock-sheet.py` → open `ebay.xlsx` → fill in Qty column (yellow) for cards in stock
2. Optional: fill Price column (yellow) for individual card overrides — blank = use script defaults
3. Run `export-ebay-csv.py` → upload CSV to Seller Hub

### Updating an existing listing

- **Add new stock / change Qty:** Revise the listing in Seller Hub directly, or re-upload the CSV using `Action = Revise` (not yet implemented in the script)
- **Add new cards to the listing:** Update `ebay.xlsx` Qty, regenerate CSV, re-upload
- **Price changes:** Can be done directly in Seller Hub without re-uploading

## eBay research findings

- **Finding API deprecated** Feb 2025 — no free sold-price data for comps
- **Marketplace Insights API** (sold data) is locked to commercial partners — not accessible
- **eBay limit:** 250 variations per multi-variation listing — me02.5 at 195 is well under
- **Postage cost:** ~$2.60 fixed per order (AusPost letter: $1.70 stamp + ~$0.90 envelope/toploader/teambag). 5mm thickness limit for letter rate.
